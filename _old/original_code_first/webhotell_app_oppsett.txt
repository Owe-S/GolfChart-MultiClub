=== Hva dette oppsettet kan brukes til ===
1. Backend / API
   - Kan kjøre PHP-basert API (REST/GraphQL) for kommunikasjon med app.
   - Appen kan hente og lagre data i MySQL-databasen.

2. Database
   - Lagring av brukere, innhold, logger m.m.
   - Egnet til små/mellomstore prosjekter (noen begrensninger i Beginner-planen).

3. Webside / Dashboard
   - Kan lage adminpanel/dashbord på owestangeland.no.
   - Bygges enten i WordPress eller egen PHP/HTML/JS-løsning.

4. App (frontend)
   - Selve appen må utvikles i f.eks. Flutter, React Native eller native kode.
   - Distribueres via Google Play og Apple App Store.
   - Appen kommuniserer med backend/API på webhotellet.

=== Viktige begrensninger ===
- Ingen Node.js/Python-støtte på webhotellet (kun PHP).
- Begrenset ytelse – ikke optimalt for mange samtidige brukere.
- Selve mobilappen publiseres ikke fra webhotellet (kun backend/database).

=== Konklusjon ===
✅ Ja – oppsettet kan brukes til å bygge og drifte backend (API + database) for en app.
❌ Nei – selve mobilappen kan ikke publiseres direkte fra webhotellet.
Mobilappen må bygges i et app-utviklingsverktøy og lastes opp til app-butikker.
